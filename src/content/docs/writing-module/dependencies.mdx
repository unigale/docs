Dependencies allow a module to require other modules to be enabled before it can function.

## Enabling Dependencies

To declare dependencies, a module should implement the [`UniGale\Foundation\Contracts\HasDependencies`](https://github.com/unigale/framework/blob/main/packages/Foundation/Contracts/HasDependencies.php) interface alongside [Module](/modules).

### Required Method

```php
public function dependencies(): array;
```

* `dependencies()` should return an array of identifiers for modules that must be enabled first.


<Aside type="tip" title="Constraints" icon="approve-check">
* A module cannot be enabled unless all of its dependencies are satisfied.
* All dependent modules must be enabled to activate this module.
</Aside>

### Example: Conditional Activation

This example ensures `BrandingModule` cannot be activated unless `core::execution-platform` is enabled:

```php
use UniGale\Foundation\Contracts\Module;
use UniGale\Foundation\Contracts\HasDependencies;
use Illuminate\Support\ServiceProvider;

class BrandingModule extends ServiceProvider implements HasDependencies, Module
{
    // ...
    public function dependencies(): array
    {
        return ['core::execution-platform'];
    }
}
```
