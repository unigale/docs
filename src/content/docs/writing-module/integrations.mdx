Integrations let modules communicate, register resources in one another, or extend shared components such as admin panels, workflows, or templates.
<Aside>
* Using integration implementations ensures that calls to override or extend an external module are not executed if the targeted module is not enabled.
* Maintains consistency across all modules and provides information about hierarchical module integrations and their supports.
</Aside>

## Registering Integrations
<Aside type="tip" icon="comment">
* You can list all active integrations for a module using the `module:status` command or via the Administration panel.
</Aside>

To enable integrations, your module should implement the [`UniGale\Foundation\Contracts\HasIntegrations`](https://github.com/unigale/framework/blob/main/packages/Foundation/Contracts/HasIntegrations.php) interface in addition to [Module](/modules).

### Required Method

```php
public function integrations(\UniGale\Foundation\IntegrationsManager $integrations): void;
```

The `IntegrationsManager` allows you to run specific closures when a target module is **enabled** or **disabled**.

### Example: Customizing the Administration Module

This example shows how a module can override the brand name of the Administration module when it is enabled:

```php
use UniGale\Foundation\Contracts\Module;
use UniGale\Foundation\Contracts\HasIntegrations;
use UniGale\Foundation\IntegrationsManager;
use Illuminate\Support\ServiceProvider;

class BrandingModule extends ServiceProvider implements HasIntegrations, Module
{
    //...
    public function integrations(IntegrationsManager $integrations): void
    {
        $integrations->forModule(
            identifier: 'core::administration',
            whenEnabled: function() {
                \UniGaleModules\Administration\Administration::configureUsing(function($admin) {
                    $admin->brandName('My Custom Brand');
                });
            },
            whenDisabled: function() {
                // Optional cleanup if Administration module is disabled
            }
        );
    }
}
```

### Summary

* Integrations let your module extend or modify other modules safely and dynamically.
* Use `whenEnabled` for active behavior, `whenDisabled` for cleanup, and always guard against missing modules.
