Schemas are composed of **properties**, which define the type, default value, title, and constraints of each piece of data.

## Creating a Schema

You can create a schema using the `Schema::create()` method:

```php
use Epsicube\Schemas\Schema;
use Epsicube\Schemas\Properties\BooleanProperty;
use Epsicube\Schemas\Properties\StringProperty;

$schema = Schema::create(
    identifier: 'UniqueIdentifier',
    title: 'Your Custom Schema',
    description: 'Contain all informations about',
    properties: [] // shortcuts for below ->append()
);
````


## Adding Properties

Once a schema is created, you can append properties to it:

```php
use Epsicube\Schemas\Properties\BooleanProperty;
use Epsicube\Schemas\Properties\StringProperty;
$schema->append([
    'enabled' => BooleanProperty::make()
        ->title('This Feature Enabled')
        ->default(true),

    'brand-name' => StringProperty::make()
        ->title('Brand Name')
        ->default(fn () => config('app.name')),
]);
```

Properties define the **type of data**, a **human-readable title**, and optional **default values**.
Default values can be static or computed dynamically using a closure.


## Property Types

The Schema system comes with several built-in property types:

* `ArrayProperty` — for arrays of values.
* `BooleanProperty` — true/false values.
* `FloatProperty` — decimal numbers.
* `IntegerProperty` — whole numbers.
* `ObjectProperty` — nested objects or complex structures.
* `StringProperty` — text values.

<Aside title="Tip">
You can combine properties to create **nested schemas**, reuse common definitions, and define complex structures in a maintainable way.
</Aside>
